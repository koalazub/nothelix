# Gutter-bar plugin change

## What
Rainbow left-gutter bars: each distinct cell output gets a cycling series
color (ui.virtual.output.series0..7, wrapping) so multiple results in one cell
are visually separable.

## Files
- plugin/nothelix/output-render.scm — added `series-scope` (index -> scope,
  wrapping mod 8) and `assign-cycling-bars` (flattens a list of row-groups,
  tagging every row in group N with series-scope N, dropping empty groups so
  they don't consume a color).
- plugin/nothelix/output-insert.scm — `update-cell-output` now builds distinct
  groups (stdout, output_repr, stderr, each text-plot) and runs them through
  `assign-cycling-bars`; the error path wraps its rows as one group. The store
  still keeps plain rows.
- plugin/nothelix/execution.scm — `restore-cell-outputs-on-open!` groups the
  stored text rows as one group plus each text-plot as its own group, same
  `assign-cycling-bars`.
- plugin/tests/output-render-test.scm — tests for series-scope wrap and
  assign-cycling-bars (cycling, wrap after 7, one multi-row group shares a
  scope, empty groups dropped, styled span-list rows wrapped intact).

## FFI shape (per-row item to set-output-lines-below!)
- `<string>` -> plain row, no bar
- `((text scope) ...)` -> styled span row, no bar
- `("bar" <bar-scope-or-#f> <payload>)` -> row with a gutter bar; `<payload>`
  is a `<string>` or `((text scope) ...)`. Marker string "bar" disambiguates.

## Grouping rule
stdout = one group, output_repr = one group, stderr = one group, each text-plot
= its own group; all rows within a group share one bar color. A matrix is one
output -> one color. On restore the concatenated stored text rows are one group
(sub-boundaries aren't recoverable from the store) plus each text-plot a group.

## Backward compat
Untagged rows still render bar-less on the fork. New plugin data requires the
new fork FFI; released together.
